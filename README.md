📖 1. 概要 (Overview)
本プロジェクトは、スマートフォンのタップ・ドラッグ操作に最適化された、真上見下ろし型（トップダウン）の人工生命パズルシミュレーションゲームである。

プレイヤーは限られたポイントを使い、クリーチャーの手足や感覚パーツを「パズル」のようにショップで配置・カスタマイズし、リアルタイムに変化し続ける生態系をコントロールする。手探りで歩き方を学んでいく生命体たちを、パズル感覚で改造してエサまで導くのが目的となる。

プレイヤーはクリーチャーを直接操作するのではなく、

エサ配置
環境変更
品種保存
突然変異誘導

などを通じて、間接的に進化と生態系へ介入する。

🛠️ 2. 大まかな開発ロードマップ (Project Milestones)
[-] フェーズ1：2D物理基盤の検証
目標: 2Dのトップダウン視点で、手足パーツがジョイントで繋がり、ランダムにバタバタとのたうち回る物理的な「身体の土台」を検証する。

[-] フェーズ2：リアルタイム世代交代 ＆ スマホ摂食ループ（★次に取り組むこと）
目標: スマートフォンで遊ぶための「命のサイクル」と操作の基本部分を完成させる。

やること:

[-] リアルタイム補充システム: 画面内の個体数を常に一定にキープ。死んだ個体が出たら、その場で即座に優秀な遺伝子を引継いだ新しい子供を1体補充スポーンさせる。

[-] スマホ用・エサのタップ投下: 画面をタップした場所にエサを配置する。

[-] HP代謝と2D HPバー: 頭上にHPバーを表示。時間経過や手足を激しく動かす（カロリー消費）ことでHPが減り、エサに触れると全回復。0になると死亡して世代交代へ送る。

[-] フェーズ3：感覚センサーと脳（AI）のガッチャンコ
目標: ランダム運動を卒業し、ニューラルネットワークに手足を自律制御させて学習を開始する。

やること:

[-] 2Dエサセンサー: 最も近いエサへの「方向（ベクトル）」と「距離」を計算し、AIの入力に渡す。

[-] 脳と手足の接続: AIの計算結果を各手足のモーター速度に流し込み、エサに向かって進むようリアルタイムで遺伝的アルゴリズム（進化）を回す。

[-] フェーズ4：パーツの動的遺伝 ＆ ショップ改造パズルシステム
目標: ゲームの核心である「パズル要素」と「ショップ」を実装する。

やること:

[-] 動的パーツ生成: 遺伝子データ（手足の数など）を読み込んで、ゲーム開始時に手足ノードを動的に組み立てる仕組み。

[-] ショップ改造UI: スマホ画面上で個体をタップ選択し、エサやポイントを消費して「手足を増やす/減らす」「感覚（目・耳・第六感）を追加する」など、パズル感覚で身体を組み替えるエディタUIの実装。

[x] フェーズ5：環境ビルド ＆ 系統誘導システム
目標: 
プレイヤーが単に個体を改造するだけでなく、「新しい進化のルール」を集めて組み合わせ、独自の環境（スポーンポイント）をパズルのように構築して生態系をコントロールする。

やること:

[x] 新ルールのアイテム化と収集 (Collectible Evaluations):
フィールド上に「スピードの欠片」「ジャンプの欠片」などの特殊アイテムを出現させる。クリーチャーがこれらを拾うことで、新しい「評価基準（Evaluation Rule）」がプレイヤーの手持ち（インベントリ）としてアンロックされる。

[x] カスタムスポーンポイントのビルド:
アンロックされた複数の評価基準（例：「エサ探索」＋「ジャンプ力」）をプレイヤーが自由に組み合わせ、新しいスポーンポイントをフィールドに設置・カスタマイズできるシステムを作る。

[x] 個体保存と血統の割り当て:
プレイヤーが気に入った優秀な個体（親）を保存し、構築したカスタムスポーンポイントにセットすることで、「指定した環境ルールで、指定した親から進化し続ける独自の血統」を生み出せるようにする。

[x] 生態系環境ギミック:
地形ギミック（壁、段差、空中足場など）とカスタムスポーンポイントを組み合わせることで、「この地形を乗り越えるための進化」をプレイヤー自身がデザインする。


実装メモ（Phase 5）:

- EvaluationRule は UnlockableEvaluationRule / EvaluationInventory / EvaluationRuleUnlock により、フィールド上の欠片Prefabからアンロック可能。
- SpawnEvolutionProfile / SpawnRuleSet / SpawnBehaviorSettings / CustomSpawnPoint により、スポーン地点ごとに評価ルール、重み、突然変異バイアス、血統ソースを保持可能。
- SavedCreatureData / CreatureArchive / LineageData により、有望個体のGenome・色・名前・系統ID・評価履歴をRuntime保存し、SpawnPointの祖先として再利用可能。
- EnvironmentZone / BridgeTraversalReward / HazardPenaltyZone / FrictionZone と EnvironmentInteractionRule により、Trigger/Colliderベースの環境ギミックを評価へ接続可能。
- サンプルとして Assets/EvolutionBuild/SpawnProfiles と Assets/EvolutionBuild/SpawnPoints に「高速移動型スポーン」「探索型スポーン」を追加済み。

[ ] フェーズ6：系統保存と生態系誘導システム

目標:
プレイヤーが「有望な個体」を保存・保護・繁殖誘導できるようにし、生態系へ間接的に介入できるようにする。

やること:

[ ] 個体保存システム:
プレイヤーが気に入った個体をつまんで保存し、いつでも参照可能にする。

[ ] 系統スポーンポイント:
保存した個体の遺伝子をベースにした個体が継続的に出現するスポーンポイントを設置できるようにする。

[ ] 系統の可視化:
色・模様・名前などにより、どの個体がどの系統に属しているか分かるようにする。

[ ] 生態系環境ギミック:
橋・穴・摩擦床・危険地帯などを配置し、環境によって有利な進化が変化するようにする。

🔬 3. スマホ・パズル要素の詳細仕様
スマホ最適化物理:
重力はなく、水中のような摩擦（Drag）空間。指でのタップや、クリーチャーを直接ドラッグして動かせる「神の手」操作との親和性を高くする。

パズル要素（身体のビルド）:
「手足をここに増やすと、AIがこういう泳ぎ方を編み出す」「目を追加するとエサを発見しやすくなるが、消費カロリー（HPの減り）が激しくなる」といった、メリット・デメリットをパズルのように考えて身体を組み立てる戦略性をゲームのメインの面白さとする。
